# mylife 设计文档

## 1. 产品定位

`mylife` 是面向个人工作的时间管理软件，目标替代当前基于 Notion 的任务管理方式，降低操作复杂度，强化任务生命周期可视化。

核心原则：

- 纯本地优先，断网可用。
- 简单任务路径极短：`创建任务 -> 点击完成`。
- 复杂任务支持主任务、子任务、孙子任务树。
- 每个任务状态变更都记录到生命周期时间轴。
- 时间轴可生成甘特图，按状态颜色区分。
- 线上能力未来只作为同步增强，不侵入本地主功能。

## 2. v1 范围

本期以 `Mac 客户端优先` 为目标，技术路线为：

- `Tauri` 作为桌面壳
- `React` 作为前端界面
- `Go` 作为本地业务核心
- `SQLite` 作为本地数据存储

`v1` 必须包含：

- 工作空间管理
- 今天视图
- 任务树管理
- 四种默认状态：`未开始`、`进行中`、`已完成`、`废弃`
- 状态可扩展
- Markdown 任务文档
- 文档中的 `[]` 待办自动收集为任务树节点
- 单次提醒与重复提醒
- 按真实状态事件生成甘特图
- 桌面置顶小窗，仅展示今天进行中的任务
- 任务操作后的 `5 秒撤销 toast`
- 主题切换
- GitHub 自动生成 Mac 安装包

## 3. 信息架构

主窗口采用 `左 / 中 / 右` 三栏结构。

### 3.1 左栏

上半区：

- 月历组件
- `v1` 中仅作为日期过滤器使用
- 点击某天后，切换主视图任务范围

下半区：

- 工作空间列表
- 每个工作空间独立管理任务，相当于独立租户
- 用户可在空间之间切换

### 3.2 中栏

中栏为主工作区，默认落点为 `今天`。

布局规则：

- 按状态分组展示任务
- 每个状态区块可展开 / 折叠
- 支持快速新增任务
- 支持拖拽任务切换状态
- 支持点击直接完成任务
- 任务卡显示状态、提醒、今天标识等关键信息

默认分组：

- 未开始
- 进行中
- 已完成
- 废弃

### 3.3 右栏

右栏由两个页签组成：

1. `任务详情`
2. `甘特图`

#### 任务详情页签

- 展示当前选中任务的基础信息
- 展示 Markdown 文档
- 展示从 Markdown `[]` 解析出的任务树
- 展示提醒设置

#### 甘特图页签

- 展示当前任务树或当前筛选范围内任务的甘特图
- 支持按任务状态筛选
- 状态以颜色区分
- 时间轴基于真实状态事件生成

## 4. 核心交互

### 4.1 简单任务

用户路径：

1. 快速新增任务
2. 进入列表
3. 点击完成

要求：

- 不强迫填写复杂字段
- 默认最少操作完成闭环

### 4.2 复杂任务

复杂任务支持层级：

- 主任务
- 子任务
- 孙子任务

每个节点都必须具备：

- 独立状态
- 独立生命周期记录
- 独立提醒能力

### 4.3 Markdown 与任务双向绑定

右侧文档中的 `[]` 视为真实待办任务，不是纯文本标记。

规则：

- `- [ ] xxx` 创建未完成任务节点
- `- [x] xxx` 标记对应任务已完成
- 文档勾选变化要同步到任务状态
- 任务状态变化也要回写文档复选框
- 解析后在右侧展示树状结构

### 4.4 状态切换撤销

每次任务状态切换后，软件内右下角弹出 `toast`：

- 展示新状态结果
- 提供 `撤销` 操作
- 持续 `5 秒`
- 主窗口与桌面小窗都适用

## 5. 状态系统

默认状态：

- `未开始`
- `进行中`
- `已完成`
- `废弃`

要求：

- 状态定义可扩展
- 每个状态都绑定独立颜色
- 前端所有任务展示与甘特图都复用同一颜色映射

默认颜色建议：

- 未开始：灰蓝
- 进行中：琥珀
- 已完成：绿色
- 废弃：砖红

## 6. 甘特图设计

甘特图不是计划排期工具优先，而是 `生命周期实录` 优先。

规则：

- 每次状态变更都记录真实发生时间
- 甘特图按状态事件切分连续时间段
- 同一任务可以有多个不同状态色块
- 可按状态筛选显示
- 任务树中的子任务独立成行
- 主任务与子任务保持层级可读性

`v1` 不强制引入计划开始时间和截止时间作为核心模型，避免产品复杂度失控。

## 7. 提醒设计

`v1` 支持：

- 单次提醒
- 重复提醒

重复提醒至少支持：

- 每天
- 每周
- 自定义按周多选

提醒展示规则：

- 主窗口任务卡可见提醒摘要
- 右侧详情区可编辑提醒时间和重复规则
- 到点触发系统通知
- 应用内同时高亮任务

## 8. 桌面小窗

桌面小窗为 `置顶轻交互窗口`。

用途：

- 常驻桌面
- 聚焦展示今天正在做的任务

默认筛选条件：

- `今天`
- `进行中`

支持操作：

- 直接完成
- 搁置或恢复进行
- 打开任务详情

不在小窗中支持：

- 全量编辑
- 拖拽排序
- 新增复杂任务

## 9. 主题设计

首版支持主题切换，至少提供多套主题色。

要求：

- 主题切换入口位于主窗口顶栏
- 状态色与主题基础色分离
- 切换主题后不影响状态识别

风格方向：

- 参考 Notion Calendar 的清爽秩序感
- 图标使用扁平化风格
- 避免过重的企业后台气质
- 保持任务信息密度与阅读舒适度平衡

## 10. 前端视觉结论

确认采用 `今天优先三栏台` 方案：

- 左栏承载日期与工作空间
- 中栏承载状态任务流
- 右栏承载详情与甘特图

该方案吸收：

- 看板式拖拽效率
- 文档式任务拆解
- 时间轴式生命周期表达

但不让甘特图占据主工作流中心，避免日常使用过重。

## 11. 非目标

`v1` 不包含：

- 强依赖联网能力
- 云端账号体系
- 团队协作
- 像完整项目管理软件那样的重排期逻辑

## 12. 发布与交付

`v1` 完成后，代码推送到 GitHub 时需要具备生成 Mac 安装包的能力。

要求：

- 在 GitHub 上可直接触发构建
- 输出可下载的 Mac 安装包
- 默认生成 `dmg` 安装产物
- 安装产物作为 GitHub Actions artifact 或 GitHub Release 附件保存
- 本地开发流程不依赖手工打包

## 13. 设计决策摘要

- 平台优先级：`Mac 优先`
- 默认主视图：`今天`
- 数据策略：`纯本地优先`
- 核心界面：`三栏结构`
- 文档待办：`[] 直接映射真任务`
- 甘特图语义：`生命周期实录`
- 提醒能力：`单次 + 重复`
- 小窗定位：`置顶轻交互`
- 交互补强：`主题切换 + 5 秒撤销 toast`
