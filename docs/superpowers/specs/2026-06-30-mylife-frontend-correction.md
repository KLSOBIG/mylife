# mylife 前端设计纠偏

更新时间：`2026-06-30`

本文档覆盖对当前前端实现的纠偏要求。优先级高于此前原型中的临时占位表现。

## 问题结论

当前页面存在以下根问题：

- 左侧不是完整月历，只是几天按钮，违背 `日历` 定义
- 主题切换只有几个绿色点，没有明确组件语义
- 中间主区域横向铺太开，空间浪费严重
- 状态列缺少拖拽，是核心需求缺失
- 任务创建依赖 `输入 + 保存`，不符合轻量任务流
- 任务卡交互贫弱，缺少层级、拖拽反馈、状态切换入口
- 右侧标签页是假按钮堆叠，不是清晰页签结构
- Markdown 没有编辑器体验，且文档区不占主导
- 文档内容直接输出原文字符串，没有结构化渲染
- 整体视觉与 Notion Calendar 的密度、克制、导航结构差距过大

## 新设计目标

- 视觉方向：靠近 `Notion Calendar` 的桌面信息架构与密度，不照抄样式
- 首轮预览：先产出可编译 `H5` 页面，确认交互和布局
- 再封装到 `Mac`
- 每个核心区必须可用，不允许占位式假界面

## 布局纠偏

### 左侧

- 上半区必须展示完整当月月历
- 月历支持上月/下月日期弱化显示
- 今天高亮
- 选中日期高亮
- 下半区展示工作空间列表
- 风格参考 Notion Calendar 左栏：紧凑、连续、信息密度高

### 中间

- 主区域改为 `纵向状态泳道`
- 每个状态是一段可折叠列表，不是四个大空卡片并排平铺
- 列表支持：
  - 拖拽改状态
  - 拖拽排序
  - 点击进入详情
  - 快速完成
  - 快速推进到下一状态
- 新建任务不再依赖显式 `保存` 按钮
- 用户输入标题后，失焦、回车、切换区域都应自动保存

### 右侧

- 顶部为真正的页签：`任务详情` / `甘特图`
- `任务详情` 下，Markdown 编辑区是最大区域
- Markdown 应被正常渲染，而不是直接输出原文字串
- 文档中的 `[]` 要在编辑器与任务树两侧可见
- 提醒、状态、父子关系是次级信息区
- 甘特图必须挂在真实日期轴上，不允许只画几个脱离日期的短横条

## 交互纠偏

### 任务创建

- 默认支持内联创建
- 空白输入不保存
- 有内容即自动落盘

### 拖拽

- 拖动任务卡进入不同状态区时，直接触发状态切换
- 拖动中显示落点反馈
- 拖动结束弹 `5 秒撤销 toast`

### 任务卡

- 必须展示最少信息：
  - 标题
  - 状态
  - 提醒
  - 子任务数
  - 是否今天
- 卡片支持 hover、active、selected 三态

### 标签页

- 只保留一组页签
- 页签切换不允许重复按钮堆叠

## Markdown 纠偏

- 右侧主区域必须可编辑
- 默认呈现渲染后的文档，而不是原始字符串
- 需要在 `编辑 / 预览` 间切换，或采用即时渲染编辑器
- 文档中的 checkbox 与任务系统双向同步

## H5 预览要求

- 每轮大改先提供可运行 `H5`
- 在浏览器确认后，再同步桌面壳
- H5 至少覆盖：
  - 左侧完整月历
  - 中间纵向状态拖拽区
  - 右侧 Markdown 编辑/预览
  - 真页签
  - 日期轴甘特图
  - 撤销 toast

## 参考

参考 Notion 官方页面与用户提供截图。重点吸收：

- 左栏完整月历与账号/空间层次
- 顶栏稀疏但明确的导航
- 主内容区高密度网格
- 控件克制，不用廉价色块充数
