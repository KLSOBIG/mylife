# mylife 组件规范

更新时间：`2026-06-30`

本规范用于约束 H5 与 Mac 客户端共用前端组件。目标：不再出现临时拼凑按钮、空白大块、假标签页、假编辑区。

## 1. 左侧栏

### 1.1 月历面板

结构：

- 顶部：月份标题、上一月/下一月切换
- 中部：周标题 `周日 ~ 周六`
- 主体：`7 x 6` 日期网格
- 底部：日期说明或当日筛选状态

规则：

- 当月日期正常展示
- 上月/下月补位日期弱化显示
- 今天有独立高亮
- 选中日期有实心高亮
- 日期点击后，刷新中间与右侧上下文

### 1.2 工作空间列表

结构：

- 标题
- 工作空间列表
- 当前空间高亮
- 可扩展入口

规则：

- 与月历共用左栏，但视觉分区清楚
- 列表行高紧凑，不做大卡片

## 2. 中间主区

### 2.1 今日头部

结构：

- 左：`今天` 标题 + 当前日期说明
- 中：快速筛选或日期上下文
- 右：主题切换、快速新建

规则：

- 主题切换使用明确按钮，不允许只有色点悬空
- 快速新建入口固定在头部右侧

### 2.2 状态泳道

结构：

- 纵向堆叠，不横向平铺
- 每条泳道有：
  - 状态名
  - 任务数
  - 折叠按钮
  - 新建入口

规则：

- 一次只滚动一个主内容区
- 泳道之间间距紧凑
- 单泳道为空时展示轻空态，不占大面积

### 2.3 任务卡

最小字段：

- 标题
- 状态标签
- 提醒时间
- 子任务数
- 今天标记

交互：

- 点击：打开右侧详情
- hover：显示快捷动作
- 快捷动作：
  - 完成
  - 下一状态
  - 更多
- 支持拖拽排序
- 支持拖拽跨泳道改状态

视觉：

- 平面卡片
- 左侧状态色条
- 卡片阴影极弱
- 选中态清晰，但不过度高亮

### 2.4 内联创建器

结构：

- 轻量输入行，嵌在对应泳道顶部或头部

规则：

- 输入后按回车保存
- 失焦保存
- 空白丢弃
- 不使用额外 `保存任务` 大按钮

## 3. 右侧详情区

### 3.1 顶部页签

页签只有一组：

- `任务详情`
- `甘特图`

规则：

- 使用真 tab 结构
- 不允许再出现重复按钮行

### 3.2 任务详情页

结构顺序：

1. 任务标题与状态
2. Markdown 主编辑区
3. 文档解析出的任务树
4. 提醒与元信息

规则：

- Markdown 编辑区必须是右侧最大区域
- 默认支持编辑
- 支持预览或即时渲染
- 文档中 `[]` 实时映射到任务树

### 3.3 甘特图页

结构：

- 筛选条
- 图例
- 甘特时间轴主体

规则：

- 状态筛选清楚
- 层级缩进清楚
- 颜色和任务状态完全一致

## 4. Toast

结构：

- 文案
- 撤销按钮
- 倒计时弱提示

规则：

- 固定右下角
- 5 秒自动关闭
- 主窗口与小窗统一表现

## 5. 桌面小窗

结构：

- 顶部标题
- 今天进行中任务列表
- 单卡片轻交互按钮

规则：

- 只做轻操作
- 不做复杂编辑
- 不做拖拽

## 6. 反模式

禁止出现：

- 只有几个日期按钮却叫月历
- 只有色点却叫主题切换
- 两排按钮却叫标签页
- 原始 Markdown 字符串直出
- 任务创建依赖显式保存
- 大片空白占据主视觉
